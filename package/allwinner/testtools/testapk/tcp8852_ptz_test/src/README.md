# TCP8852 PTZ测试程序使用说明

## 程序简介

`tcp8852_ptz_test.c`是一个用于测试TCP8852芯片的PTZ（云台控制）功能的测试程序。该程序支持选择不同的通道和电机动作进行控制，可用于测试和调试TCP8852驱动。

## 使用方式

### 基本语法

```bash
tcp8852_ptz_test [选项]
```

### 命令行选项

- `-h`: 显示帮助信息
- `-c [通道号]`: 指定要控制的通道，支持的通道包括：1, 2, 12, 3, 4, 34
- `-m [电机动作]`: 指定电机动作，支持的动作包括：
  - 0: OPEN_CIRCUIT（开路）
  - 1: FOREWARD（正转）
  - 2: INVERSION（反转）
  - 3: BRAKE（刹车断电）

## 功能说明

程序支持多种控制模式，根据不同的命令行参数进入相应的测试模式：

1. **初始化自动控制模式**：通过`-a`选项进入
2. **初始化直接控制模式**：通过`-d`选项进入
3. **初始化通道测试模式**：通过`-t`选项进入
4. **初始化IRCUT测试模式**：通过`-i`选项进入

## IRCUT操作注意事项

**重要提示**：在IRCUT操作中，执行FOREWARD（正转）或INVERSION（反转）动作后，必须执行BRAKE（刹车断电）操作，以确保电机安全停止并断电。

### IRCUT正确操作流程示例：

```bash
# 执行IRCUT正转
./tcp8852_ptz_test -i -m 1

# 执行IRCUT刹车断电（必须执行）
./tcp8852_ptz_test -i -m 3
```

## 使用示例

### 1. 选择通道1并执行正转操作

```bash
./tcp8852_ptz_test -c 1 -m 1
```

### 2. 选择通道12并执行反转操作

```bash
./tcp8852_ptz_test -c 12 -m 2
```

### 3. 选择通道34并执行刹车断电操作

```bash
./tcp8852_ptz_test -c 34 -m 3
```

### 4. 进入IRCUT测试模式并执行正转后刹车

```bash
# 进入IRCUT测试模式并正转
./tcp8852_ptz_test -i -m 1

# 执行刹车断电
./tcp8852_ptz_test -i -m 3
```

## 注意事项

1. 请确保设备已正确连接并有权限访问TCP8852设备节点
2. 使用前请确认目标通道和动作的正确性，避免不当操作导致设备损坏
3. IRCUT操作完成后务必执行BRAKE操作以保护电机
4. 测试过程中如发现异常，请立即停止测试并检查连接和配置

## 支持的通道列表

- CH1: 通道1
- CH2: 通道2
- CH12: 通道12（组合通道）
- CH3: 通道3
- CH4: 通道4
- CH34: 通道34（组合通道）

## 支持的电机动作列表

- OPEN_CIRCUIT (0): 开路状态，电机不工作
- FOREWARD (1): 正转
- INVERSION (2): 反转
- BRAKE (3): 刹车断电，停止电机并断电
